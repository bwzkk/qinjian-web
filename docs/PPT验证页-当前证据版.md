# 亲健 PPT 验证页（当前证据版）

> 用途：这一页是给比赛 PPT 直接用的，不是内部开发记录。内容尽量只放已经核过、现场能讲清、评委一眼能懂的证据。

## 一、推荐页标题

- 系统验证与工程可信度
- 测试验证与演示稳定性
- 工程验证与落地可行性

推荐直接用：`系统验证与工程可信度`

## 二、可直接放进 PPT 的页面文案

### 版本 A：偏正式答辩版

#### 系统验证与工程可信度

- 自动化测试：后端当前复核已收集 `223` 条测试，覆盖认证安全、配对访问控制、打卡报告、关系智能、隐私治理等核心模块；前端工具层 `101/101` 自动化测试全部通过。
- 构建验证：Vue 3 前端生产构建通过，说明项目不仅能在开发环境运行，也具备正式部署产物。
- 真实环境验证：仓库保留了冒烟与轻量压测脚本，最终答辩前可重新生成“注册、登录、创建关系、加入关系、记录、简报、修复协议、演示页”全链路证据。
- 页面稳定性：最终提交前建议用当天运行结果补入 `console/page/request` 异常统计，避免沿用旧日志。
- 压力验证：仓库保留轻量压测脚本，可用于复核关键读取链路的响应稳定性。
- 隐私与安全：系统已实现签名访问、隐私审计、删除申请、安全响应头和上传目录非公网直出，不是只停留在隐私声明层面。

## 五、答辩时别主动说的内容

- 不要把基础压测结果直接讲成“高并发能力很强”。更稳妥的表述是“已完成基本稳定性验证”。
- 不要把“压测通过”讲成“高并发能力很强”。更稳妥的表述是“已做基本稳定性验证”。

## 七、这一页对应的证据来源

- 自动化测试抽样：`backend/` 下核心测试分组复核

- 前端工具测试：`web-vue3/` 下 `npm test`
- 前端构建：`web-vue3/` 下 `npm run build`
- 真实环境冒烟：`output/frontend_real_env_smoke.py`
- 压力验证：`output/api_light_pressure.py`
