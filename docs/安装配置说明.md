# 安装配置说明

## 文档信息

| 项 | 说明 |
| --- | --- |
| 文档类型 | 安装与部署说明 |
| 适用对象 | 开发人员、测试人员、运维人员、交付接收方 |
| 适用范围 | 本地开发、容器化部署、服务器部署、环境变量配置 |
| 更新依据 | 以当前仓库代码、脚本和配置文件为准 |

## 1. 概述

本文档用于指导亲健项目的安装、配置、启动和基础排查。

当前代码口径如下：

- 前端开发服务默认端口：`3001`
- 后端本地联调默认端口：`8010`
- Docker Compose 对外访问入口：`80`
- 健康检查路径：`/api/health`

说明：

- 前端端口来自 `web-vue3/vite.config.js`。
- 后端联调端口可参考 `tmp/start_manual_backend.ps1`。
- 容器部署时，Nginx 对外暴露 `80` 端口，后端容器内部监听 `8000`。

## 2. 适用场景

| 场景 | 推荐方式 | 说明 |
| --- | --- | --- |
| 页面开发与接口联调 | 本地开发模式 | 启动快，便于修改前后端代码 |
| 验证完整运行链路 | Docker Compose | 更接近部署态 |
| 对外服务发布 | 服务器部署 | 配合 `.env`、Docker Compose 和 Nginx 使用 |
| 仅做答辩或界面演示 | 前端 + `/demo` | 不依赖真实账号即可演示核心流程 |

## 3. 前提条件

建议准备以下基础环境：

| 组件 | 建议版本 | 用途 |
| --- | --- | --- |
| Python | 3.12 | 与 `backend/Dockerfile` 保持一致 |
| Node.js | 18 LTS 或以上 | 前端依赖安装与 Vite 开发 |
| npm | 9 或以上 | 前端依赖管理 |
| Docker / Docker Compose | 最新稳定版 | 全栈容器部署 |
| PostgreSQL | 16 | 生产环境推荐数据库 |
| Redis | 7 | 手机验证码与登录风控推荐存储 |

说明：

- 本地快速联调可直接使用 SQLite，不强制依赖 PostgreSQL。
- 仅做页面演示时，可先跳过数据库和 AI 配置，直接使用 `/demo` 入口。

## 4. 目录与配置文件

安装部署过程中最常使用的文件如下：

- 根目录 `.env.example`：Docker Compose 和服务器部署环境变量模板。
- `backend/.env.example`：本地直接运行 FastAPI 的环境变量模板。
- `docker-compose.yml`：容器编排入口。
- `nginx.conf`：Nginx 代理和安全响应头配置。
- `web-vue3/vite.config.js`：前端端口和接口代理规则。
- `deploy_current_workspace.py`：一键部署脚本。

## 5. 本地开发模式

### 5.1 步骤一：准备后端运行环境

```powershell
cd C:\Users\colour\Desktop\亲健web\backend
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

建议先将 `backend/.env` 调整为适合本地联调的最小配置：

```env
DATABASE_URL=sqlite+aiosqlite:///./qinjian.db
FRONTEND_ORIGIN=http://127.0.0.1:3001
SECRET_KEY=0123456789abcdef0123456789abcdef
ENABLE_API_DOCS=true
PHONE_CODE_STORE=memory
```

说明：

- 使用 SQLite 可降低首次启动门槛。
- `ENABLE_API_DOCS=true` 便于本地查看调试接口文档。
- 如果需要验证 Redis 相关逻辑，再切换 `PHONE_CODE_STORE=redis`。

### 5.2 步骤二：启动后端服务

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8010
```

启动完成后，可验证以下地址：

- 健康检查：`http://127.0.0.1:8010/api/health`
- Swagger 文档：`http://127.0.0.1:8010/docs`

### 5.3 步骤三：准备并启动前端服务

```powershell
cd C:\Users\colour\Desktop\亲健web\web-vue3
npm install
npm run dev
```

启动完成后，可通过以下地址访问：

- 前端开发服务：`http://127.0.0.1:3001`

说明：

- 当前前端会将 `/api` 请求自动代理到 `http://127.0.0.1:8010`。
- 如果后端未启动，登录、上传、报告等真实接口会报错，但 `/demo` 演示页仍可用。

### 5.4 步骤四：结果验证

建议至少完成以下检查：

1. 打开 `http://127.0.0.1:3001/demo?route=/`，确认前端可正常加载。
2. 访问 `http://127.0.0.1:8010/api/health`，确认后端返回健康状态。
3. 在前端页面触发一次接口调用，确认代理链路正常。

## 6. Docker Compose 部署模式

```env
QWEN_ASR_REALTIME_MODEL=qwen3-asr-flash-realtime
```

### 6.2 步骤二：构建前端产物

```powershell
cd C:\Users\colour\Desktop\亲健web\web-vue3
npm install
npm run build
```

### 6.3 步骤三：启动容器

```powershell
cd C:\Users\colour\Desktop\亲健web
docker compose up -d --build
```

### 6.4 步骤四：结果验证

启动完成后，可验证：

- Web 入口：`http://localhost`
- 健康检查：`http://localhost/api/health`

可使用以下命令查看状态：

```powershell

docker compose logs -f
docker compose logs -f backend
```

## 7. 核心配置项说明

### 7.1 根目录 `.env`

根目录 `.env` 用于 Docker Compose 和服务器部署。关键配置项如下：

| 变量 | 说明 |
| --- | --- |
| `DB_PASSWORD` | PostgreSQL 密码，生产环境必须替换默认值 |
| `SECRET_KEY` | JWT 与应用签名密钥，建议使用 32 字节以上随机字符串 |
| `FRONTEND_ORIGIN` | 前端访问域名，用于 CORS 和部署口径 |
| `APP_TIMEZONE` | 应用时区，默认 `Asia/Shanghai` |
| `AI_API_KEY` / `AI_BASE_URL` | 通用 OpenAI 兼容网关配置 |
| `SILICONFLOW_API_KEY` | 硅基流动兼容配置 |
| `QWEN_ASR_API_KEY` | Qwen3 ASR 服务密钥 |
| `ASR_PROVIDER` / `REALTIME_ASR_PROVIDER` | 语音服务提供方 |
| `PHONE_CODE_STORE` | 手机验证码存储，生产建议使用 `redis` |
| `REDIS_URL` | Redis 连接地址 |

### 7.2 `backend/.env`

`backend/.env` 用于直接运行 FastAPI，本地联调最常用的配置项包括：

- `DATABASE_URL`
- `FRONTEND_ORIGIN`
- `ENABLE_API_DOCS`
- `SECRET_KEY`
- `PHONE_CODE_STORE`
- `REDIS_URL`

推荐的本地最小配置如下：

```env
DATABASE_URL=sqlite+aiosqlite:///./qinjian.db
PHONE_CODE_STORE=memory
FRONTEND_ORIGIN=http://127.0.0.1:3001
```

## 8. 服务器部署

### 8.1 一键脚本部署

```powershell
cd C:\Users\colour\Desktop\亲健web
python deploy_current_workspace.py --host <服务器IP> --username root --password <密码>
```

脚本默认会执行以下操作：

1. 本地构建 `web-vue3/dist`。
2. 打包 `backend`、`web-vue3`、`docker-compose.yml`、`nginx.conf`。
3. 上传到远端部署目录。
4. 校正 `FRONTEND_ORIGIN`。
5. 远端执行 `docker compose up -d --build`。

### 8.2 手动部署

```powershell
cd C:\Users\colour\Desktop\亲健web\web-vue3
npm install
npm run build

ssh root@<服务器IP>
cd /root/qinjian
git pull
docker compose up -d --build
docker compose ps
```

## 9. 常见问题

### 9.1 前端可以打开，但接口全部失败

请优先检查以下项：

1. 后端是否运行在 `8010`。
2. `web-vue3/vite.config.js` 中的代理目标是否仍为 `8010`。
3. `backend/.env` 中 `FRONTEND_ORIGIN` 是否设置为 `http://127.0.0.1:3001`。

### 9.2 容器启动后，后端立即退出

请优先检查以下项：

1. `SECRET_KEY` 是否仍使用弱默认值。
2. `DB_PASSWORD` 是否仍使用弱默认值。
3. `PHONE_CODE_DEBUG_RETURN`、`PHONE_CODE_TEST_POPUP_ENABLED` 是否在生产误开启。

### 9.3 图片或语音上传后无法直接通过公网链接访问

这是预期行为。当前实现默认不直接公开暴露上传目录，附件访问应通过后端签名接口完成。

### 9.4 演示模式可用，但实时语音不可用

这是预期行为。演示模式使用 `demo-mode` 身份，实时语音链路要求真实账号票据，不在演示模式下开放。

## 10. 推荐实施顺序

如果是首次接手项目，建议按以下顺序完成安装：

1. 先启动前端，确认页面基础渲染正常。
2. 再以 SQLite 启动后端，确认本地接口联调正常。
3. 通过 `/demo` 入口验证关键页面和演示数据。
4. 再切换真实登录、上传、报告等完整接口链路。
5. 最后再进行 Docker Compose 或服务器部署。
