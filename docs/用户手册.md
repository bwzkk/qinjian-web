# 用户手册

## 文档信息

| 项 | 说明 |
| --- | --- |
| 文档类型 | 用户操作指南 |
| 适用对象 | 演示人员、评审人员、测试人员、日常使用者 |
| 适用范围 | 亲健 Web 当前版本主要页面与核心操作流程 |
| 使用目标 | 帮助用户完成登录、建联、记录、查看洞察、回看归档和隐私设置 |

## 1. 概述

本文档介绍亲健 Web 当前版本的主要使用方式，重点覆盖以下链路：

1. 进入系统。
2. 建立或加入关系。
3. 完成一次记录。
4. 查看简报与干预建议。
5. 回看时间轴和隐私设置。

## 2. 功能地图

| 功能模块 | 主要页面 | 主要用途 |
| --- | --- | --- |
| 登录与账号 | `/auth` | 注册、登录、验证码登录、资料维护 |

| 简报与洞察 | `/report`、`/alignment`、`/message-simulation`、`/repair-protocol` | 查看报告、双视角、消息预演和修复建议 |
| 关系空间 | `/relationship-space/:pairId` | 查看当前关系状态、信号分和建议动作 |
| 时间轴归档 | `/timeline` | 回看记录、报告节点和归档摘要 |
| 隐私与安全 | `/privacy-security` | 设置记录保存方式、查看审计和删除申请 |

## 3. 快速开始

### 3.1 进入方式

系统支持两种进入方式：

#### 真实账号模式

适用场景：

- 正式使用
- 接口联调
- 完整功能验证

入口：

- `/auth`

支持能力：

- 账号密码注册与登录
- 手机验证码登录
- 资料修改与密码管理

#### 样例演示模式

适用场景：

- 答辩
- 路演
- 功能讲解
- 不依赖真实账号的页面展示

入口示例：

- `/demo?route=/`
- `/demo?route=/pair`
- `/demo?route=/report`
- `/demo?route=/message-simulation`

说明：

- 演示模式会自动注入独立样例身份和关系数据。
- 演示模式不会写入真实账号。
- 实时语音等依赖真实票据的能力不会在演示模式下开放。

### 3.2 首次使用建议路径

建议首次使用按以下顺序进行：

1. 登录或注册账号。
2. 在“隐私与安全”完成基础偏好设置。
3. 在“关系管理”创建邀请码或输入邀请码。
4. 建立关系后完成一次“今日记录”。
5. 查看“简报”“双视角”“发前先过一遍”“缓和建议”。
6. 在“时间轴”回看记录结果。

## 4. 主要操作说明

### 4.1 登录与账号设置

登录后，建议优先检查以下内容：

- 昵称、语言、默认入口等基础资料。
- 记录保存方式。
- 是否开启辅助整理。

在“隐私与安全”页，用户可选择两种保存策略：

- `同步到账号`：适合多设备连续使用，便于报告和回看。
- `本机优先`：适合高敏感内容，本地优先保留。

用户还可以：

- 查看最近的隐私相关操作记录。
- 查看上传访问、审计状态和删除宽限期。
- 发起账号删除申请。

### 4.2 关系管理

“关系管理”页支持查看关系状态、发起关系和加入关系。

#### 发起关系

系统会生成一组邀请码。用户可执行以下操作：

1. 复制邀请码发送给对方。
2. 等待对方输入邀请码并选择关系类型。
3. 在确认后正式建立关系。

#### 加入关系

输入对方邀请码后，系统并不会立即建立关系，而是进入确认流程：

1. 输入邀请码。
2. 预览关系信息并选择关系类型。
3. 提交给对方确认。
4. 对方确认后正式生效。

#### 关系维护

建立关系后，用户可继续执行：

- 查看关系状态和关系空间。
- 修改对方备注名。
- 发起关系类型变更。
- 发起断绝关系申请，并选择是否给对方挽留机会。
- 若对方选择挽留，可查看对方的首条挽留说明与追加留言，再决定接受挽留或仍然断绝。
- 若 24 小时内未回应，或挽留阶段 12 小时内未完成处理，系统会自动解除关系。

### 4.3 今日记录

“今日记录”页提供两种记录方式：

- `写下来`
- `边说边整理`

#### 写下来

适合已经有较清晰表达、希望手动整理当天事件的场景。主要操作包括：

- 填写当天发生的事或原话。
- 选择情绪标签，可补充自定义情绪。
- 记录整体感受分、互动频率、谁先开口、是否深聊、是否完成约定。
- 上传图片或语音作为补充材料。

#### 图片与语音补充

在记录页可上传：

- 图片
- 语音

系统当前支持：

- 图片分析：输出场景摘要、情绪线索、风险标签和保存建议。
- 语音转写：将语音整理为文字，并可带回到记录内容中。

说明：

- 在“本机优先”模式下，系统更倾向只保留分析结果，而不是自动长期保存原图。

#### 边说边整理

适合不想先打字、想先把事情说出来的场景。用户可以：

- 先说一句，再继续整理。
- 新开一个话题。
- 回听上一条系统回复。

### 4.4 简报与洞察

记录完成后，可继续查看系统生成的整理结果。

#### 简报

“简报”页支持查看：

- 每日报告
- 周报
- 月报
- 历史报告

典型输出包括：

- 健康分
- 摘要判断
- 关键观察点
- 建议动作

#### 双视角

“双视角”页会从双方理解差异出发，输出：

- 共享叙事
- 我方视角摘要
- 对方视角摘要
- 最容易错位的点
- 建议桥接动作

适用场景：

- 需要理解“我们为什么会越说越错位”时。

#### 发前先过一遍

该功能用于消息预演。系统会输出：

- 对方可能优先接收到什么
- 当前表达可能带来的影响
- 风险等级与原因
- 更稳妥的改写版本
- 建议做法和避免做法

适用场景：

- 在发送高情绪消息前先做风险检查。

#### 缓和建议

系统会根据当前风险等级输出结构化修复动作，例如：

- 先停 10 到 20 分钟
- 一次只说一个点
- 先讲感受，再讲需求
- 最后确认一个今晚就能做到的小动作

#### 建议说明

“建议说明”页用于解释系统当前的建议依据，例如：

- 风险分层
- 干预剧本
- 完成率与反馈
- 当前关注重点

### 4.5 关系空间与时间轴

#### 关系空间

关系空间用于集中查看当前每段关系的状态，主要包含：

- 关系概览或星图
- 当前信号分与趋势
- 当前建议动作
- 主题切换

适用场景：

- 做总览展示。
- 快速说明某段关系当前处于什么状态。

#### 时间轴

时间轴用于回看事件流，包括：

- 记录条目
- 报告节点
- 归档摘要
- 原文、原图、原语音的保留状态

说明：

- 当前实现支持时间轴归档导出。
- 对非本人内容，默认显示摘要占位，而不是完整原始内容。

### 4.6 扩展功能

除主链路外，系统还提供以下扩展页面：

- `关系体检`：固定题包与趋势分数。
- `依恋类型`：依恋风格与成长建议。
- `今日安排`：系统任务与手动任务。
- `关系技巧`：社区提示和建议生成。
- `关系纪念日`：重要节点留存。
- `异地关系`：异地活动和健康指数。
- `课程`、`咨询`、`会员`：延展型服务入口。

## 5. 常用场景建议

### 5.1 答辩或路演

建议按以下顺序展示：

1. 从 `/demo?route=/pair` 进入关系管理。
2. 展示邀请码和关系确认机制。
3. 打开关系空间详情页说明“当前状态”。
4. 进入“今日记录”展示写下来和边说边整理。
5. 打开“简报”“双视角”“发前先过一遍”“缓和建议”。
6. 最后以“时间轴”或“隐私与安全”收束。

### 5.2 日常使用

建议按以下路径使用：

1. 每天进入“今日记录”完成留痕。
2. 再查看“简报”或“关系空间”确认状态变化。
3. 如存在沟通风险，再使用消息预演和缓和建议。
4. 定期通过时间轴和体检页做回看。

## 6. 常见问题

### 6.1 为什么样例模式下不能实时语音？

因为样例模式不会申请真实实时语音票据，语音链路仅对真实账号开放。

### 6.2 为什么上传文件不能直接通过公网链接访问？

这是安全设计。当前实现默认要求通过签名访问接口读取附件，不直接暴露上传目录。

### 6.3 为什么输入邀请码后没有立即建立关系？

因为当前流程为“加入预览 -> 选择关系类型 -> 对方确认 -> 正式生效”，不是单步即时绑定。

### 6.4 如果只想快速体验界面，应该怎么做？

直接打开 `/demo?route=/` 或 `/demo?route=/pair` 即可，不需要注册登录。

## 7. 使用边界

系统定位是关系记录、辅助整理、风险提示和沟通支持工具，不替代：

- 医疗诊断
- 法律建议
- 正式心理治疗

如果场景已出现持续升级、羞辱、威胁、明显失控或安全风险，应优先暂停系统内沟通，转向可信支持网络或专业帮助。
