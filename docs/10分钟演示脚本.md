# 亲健 10 分钟功能演示脚本

> 目标：在 10 分钟内把“记录 -> 简报 -> 预演 -> 修复 -> 回看 -> 隐私”的完整闭环讲清楚。演示使用项目内置 demo 数据，不使用真实关系数据，也不依赖现场验证码和实时生成结果。

## 一、演示前准备

### 1. 推荐打开的页面

建议提前在浏览器中打开以下标签页，答辩时直接切换，减少等待：

| 顺序 | 页面 | 地址 |
| --- | --- | --- |
| 1 | 首页总览 | `http://localhost:5173/demo?route=/` |
| 2 | 关系管理 | `http://localhost:5173/demo?route=/pair` |
| 3 | 关系空间详情 | `http://localhost:5173/demo?route=/relationship-space/22222222-2222-4222-8222-222222222222` |
| 4 | 今日记录 | `http://localhost:5173/demo?route=/checkin` |
| 5 | 关系简报 | `http://localhost:5173/demo?route=/report` |
| 6 | 消息预演 | `http://localhost:5173/demo?route=/message-simulation` |
| 7 | 双视角对齐 | `http://localhost:5173/demo?route=/alignment` |
| 8 | 修复协议 | `http://localhost:5173/demo?route=/repair-protocol` |
| 9 | 今日安排 | `http://localhost:5173/demo?route=/challenges` |
| 10 | 时间轴 | `http://localhost:5173/demo?route=/timeline` |
| 11 | 隐私安全 | `http://localhost:5173/demo?route=/privacy-security` |

如果某些页面被登录状态影响，可使用静态跳转入口：

```text
http://localhost:5173/preview-demo.html?route=/message-simulation
```

把 `route=` 后面的路径替换成 `/report`、`/timeline`、`/privacy-security` 等即可。

### 2. 演示前检查

1. 前端已启动：`cd web-vue3 && npm run dev`。
2. 浏览器能打开 `http://localhost:5173/demo?route=/`。
3. 页面缩放建议保持 `100%`，窗口宽度尽量接近桌面演示比例。
4. 预演输入句提前准备：`你今天怎么又这么晚才回我，我真的很烦。`
5. 不现场上传真实照片、聊天记录或个人音频，涉及隐私的地方只讲机制。

### 3. 开场说明

可以先说：

“为了保护真实关系数据，也为了保证现场演示稳定，我这里使用的是项目内置 demo 数据。它对应的页面结构、功能流程和后端能力都和真实系统一致，只是把真实账号替换成了样例关系。”

## 三、逐段操作与话术

### 1. 首页总览与定位（0:00 - 0:40）

操作：

- 打开 `http://localhost:5173/demo?route=/`。
- 指向首页里的关系概览、今日状态、待处理动作或入口卡片。

讲法：

“现在看到的是亲健的首页。亲健不是一个单次问答工具，而是一套面向泛亲密关系的智能感知与维系平台。它覆盖情侣、朋友、异地和单人自我整理场景，核心闭环是记录、评估、干预、修复和回看。”

要点：

- 不要一开始就讲技术细节。
- 先让评委知道：这是一个完整产品，不是一个 AI 聊天框。

### 2. 关系管理与关系空间（0:40 - 1:40）

操作：

- 切到 `http://localhost:5173/demo?route=/pair`。
- 展示创建关系、输入邀请码、关系列表和关系类型。
- 再切到关系空间详情页。
- 指向关系状态、今日任务、关系树能量节点、当前关系提示。

讲法：

“这里是关系管理入口。系统支持创建关系、输入邀请码加入关系，也支持加入预览和对方确认，所以不是单方输入邀请码就立刻绑定。进入关系空间后，可以看到这段关系的当前状态、今日建议和关系树能量。关系树的作用是把记录、互动和任务反馈转成更直观的成长感，让用户知道今天做了什么会影响这段关系。”

要点：

- 强调“泛亲密关系”，不只情侣。
- 强调“协作确认”，避免关系被单方随意改变。
- 关系树只讲“成长反馈”，不要讲成游戏积分决定真实关系。

### 3. 今日记录（1:40 - 2:40）

操作：

- 切到 `http://localhost:5173/demo?route=/checkin`。
- 展示文字输入、图片入口、语音入口。
- 指出本地预检提示：情绪、风险、隐私线索。

讲法：

“这是今日记录页。用户可以用文字、图片或语音记录今天发生了什么。我们特别做了本地预检：在内容提交前，前端会先识别情绪倾向、风险线索和隐私线索。比如图片可以选择只保留分析结果，语音可以进入转写，这样用户不需要把所有敏感原始材料都长期暴露出来。”

要点：

- 记录不是日记功能，而是后续简报、预演、修复和时间轴的输入层。
- 图片和语音只讲能力，不现场上传隐私素材。

### 4. 关系简报（2:40 - 3:50）

操作：

- 切到 `http://localhost:5173/demo?route=/report`。
- 指向日报/周报/月报、关系摘要、关键观察、建议动作和趋势内容。

讲法：

“记录完成后，系统会把离散内容整理成日报、周报或月报。这里不是单纯给一个分数，而是把近期互动、情绪和事件整理成关键观察，再给出下一步建议。也就是说，用户看到的不只是‘我写了什么’，而是‘这段关系最近正在发生什么变化’。”

要点：

- 强调趋势和解释，不要只讲健康分。
- 如果页面有空状态或等待状态，可以说真实模式下必须有记录才生成，demo 里用于展示完整结构。

### 5. 消息预演（3:50 - 5:00）

操作：

- 切到 `http://localhost:5173/demo?route=/message-simulation`。
- 输入或展示这句话：`你今天怎么又这么晚才回我，我真的很烦。`
- 指向风险等级、风险原因、低刺激改写、注意事项。

讲法：

“这是亲健很核心的能力：聊天前预演。很多冲突不是因为用户没有真实需求，而是表达方式先变成了指责。用户可以把准备发送的话放进来，系统会结合关系状态判断这句话可能带来的风险，并给出更稳妥的改写。它不是替用户说话，而是帮用户把真正想表达的感受，从攻击性表达转成更容易被听见的表达。”

要点：

- 讲“发送前止损”，这是和普通聊天 AI 的差异。
- 不要承诺 AI 一定能解决冲突，只说降低沟通风险。

### 6. 双视角叙事对齐（5:00 - 6:10）

操作：

- 切到 `http://localhost:5173/demo?route=/alignment`。
- 指向共同版本、A 视角、B 视角、误读风险、桥接动作。

讲法：

“如果消息预演解决的是一句话怎么说，双视角对齐解决的是同一件事双方究竟怎么理解。系统会把双方记录整理为共同版本、A 视角和 B 视角，再指出可能的错位点和桥接动作。这样用户不再只停留在‘我觉得对方不理解我’，而是能看到双方叙事哪里没有对上。”

要点：

- 这里突出“关系智能层”，不是单次问答。
- 讲清楚它依赖双方记录，数据不足时会更谨慎。

### 7. 修复协议与今日安排（6:10 - 7:20）

操作：

- 切到 `http://localhost:5173/demo?route=/repair-protocol`。
- 展示步骤、不要做什么、成功信号、转介提示。
- 再切到 `http://localhost:5173/demo?route=/challenges`。
- 展示系统任务、手动任务、完成反馈。

讲法：

“当关系已经进入冲突阶段，系统会生成修复协议。它不是简单说‘多沟通’，而是拆成步骤：先暂停多久、先说什么、不要做什么、什么信号说明可以继续。再往后，今日安排页会把系统建议变成具体任务，用户也可以补充自己的手动任务。这样 AI 输出就从建议进入了行动。”

要点：

- 强调“建议 -> 任务 -> 反馈”的闭环。
- 高风险场景要讲“转向可信支持和专业帮助”，不要讲系统替代专业判断。

### 8. 时间轴回看（7:20 - 8:20）

操作：

- 切到 `http://localhost:5173/demo?route=/timeline`。
- 展示关键事件、报告节点、修复节点、标签和归档说明。

讲法：

“时间轴负责把关键事件沉淀下来。很多工具当下给了建议，但过几天就找不到前因后果；亲健会把记录、简报、预演和修复过程放进同一条回看链路里。这里还做了权限区分：记录者本人可以看原文和原媒体，对方默认只看到占位摘要。这样既能复盘，也不会把敏感内容无限扩散。”

要点：

- 时间轴是“证据链”和“可解释”的体现。
- 讲权限边界，评委通常会关心隐私。

### 9. 隐私与安全（8:20 - 9:10）

操作：

- 切到 `http://localhost:5173/demo?route=/privacy-security`。
- 指向保存方式、辅助整理开关、审计记录、删除申请或安全状态。

讲法：

“亲密关系数据非常敏感，所以我们没有只写隐私声明，而是把隐私做成了可操作页面。用户可以看到当前保存方式、辅助整理开关、审计记录和删除申请。后端也配套做了签名访问、上传归属校验、文本脱敏和生产弱配置拦截，尽量让数据治理进入真实工程链路。”

要点：

- 说“尽量降低风险”，不要说绝对安全。
- 点到签名访问、审计、删除申请即可，不展开太久。

### 10. 扩展能力与总结（9:10 - 10:00）

操作：

- 快速扫过 `milestones`、`longdistance`、`health-test`、`attachment-test`、`community` 中任意 1-2 个页面。
- 回到首页或关系空间收束。

可选入口：

```text
http://localhost:5173/demo?route=/milestones
http://localhost:5173/demo?route=/longdistance
http://localhost:5173/demo?route=/health-test
http://localhost:5173/demo?route=/attachment-test
http://localhost:5173/demo?route=/community
```

讲法：

“除了刚才这条主线，系统还扩展了异地关系、里程碑、关系体检、依恋类型和社群提示。它们共同服务一个目标：让 AI 不只是回答一次问题，而是长期支持一段关系被看见、被整理、被修复。”

最后一句：

“所以亲健和普通 AI 聊天产品最大的区别，是它把 AI 放进了完整关系闭环里：先记录真实状态，再形成评估，再帮助沟通和修复，最后还能回看和治理数据。”

## 五、风险预案

### 1. 页面加载慢

优先展示这几个已提前打开的标签：

1. 关系空间详情。
2. 关系简报。
3. 消息预演。
4. 修复协议。
5. 隐私安全。

### 2. 后端没有启动

直接使用 demo 页面和静态跳转入口。

可说：

“这里展示的是内置样例数据，目的是保护真实关系隐私，也避免现场网络影响。真实接口和测试验证已经在设计文档与验证页中列出。”

### 3. AI 结果等待太久

不要现场等待生成，直接切换到 demo 已有结果页。

可说：

“真实模式会调用模型生成，现场为了稳定展示，我们使用样例结果说明页面结构和业务闭环。”

### 4. 被问到测试证据

回答：

“当前后端 `python -m pytest --collect-only -q` 可收集 223 条测试，前端工具层 `npm test -- --run` 为 101/101 通过，前端 `npm run build` 也已通过。真实环境冒烟和轻量压测脚本保留在 `output/`，最终提交前会重新跑当天证据。”

## 六、答辩时避免这样说

- 不要说“系统可以解决所有关系问题”，应说“辅助用户降低冲突风险，形成复盘和修复路径”。
- 不要说“AI 判断绝对准确”，应说“系统提供结构化辅助建议，并保留解释和边界提示”。
- 不要说“数据绝对安全”，应说“系统通过签名访问、审计、删除申请和脱敏机制降低敏感数据风险”。
- 不要在现场展示真实聊天记录、真实照片或真实手机号。

## 七、一句话版本

如果最后只剩一句话，直接说：

“亲健最大的特点，是把生成式 AI 从一次性回答器，做成了围绕长期关系的记录、评估、干预、修复、回看和隐私治理闭环。”
