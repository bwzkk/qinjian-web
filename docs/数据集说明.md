## 1. 概述

本文档用于说明亲健项目当前使用的数据资产构成、存储位置、使用边界和安全约束。

说明：

- 本文档描述的是“项目运行与演示所依赖的数据集合”，不等同于对外发布的公开研究数据集说明。
- 当前仓库默认不包含可直接对外共享的真实用户公开训练语料。
- 仓库中的数据按用途可分为演示数据、开发联调数据、测试回归数据、运行时业务数据和交付验证产物五类。

## 2. 数据分类总览

## 3. 核心数据说明

### 3.1 演示样例数据

前端演示样例位于 `web-vue3/src/demo/fixtures.js`，是当前仓库中结构最完整、最稳定的数据样本。

主要用途：

1. 为 `/demo` 入口自动注入演示身份与关系上下文。
2. 在不依赖真实账号、真实接口返回或模型实时生成的前提下展示主要页面。
3. 支撑答辩、汇报、走查和产品讲解场景。

覆盖对象包括：

- 样例用户资料与偏好设置。
- 多段关系数据与邀请码样本。
- 今日状态、通知、关系树、关系空间、任务包。
- 每日报告、消息预演、双视角、修复协议、方法学说明。
- 关系体检题包、最新结果、趋势点。
- 时间轴归档、里程碑、异地活动和健康指数。

数据特点：

- 以人工编排为主，不对应真实线上大规模用户数据。
- 结构上覆盖“建立关系 -> 日常记录 -> 智能整理 -> 干预建议 -> 回看归档”的完整链路。
- 时间戳、文本、评分、标签已预设，适合稳定演示。
- 兼容关系模式和单人模式页面逻辑。

### 3.2 运行时业务数据

项目在真实运行时会产生以下业务数据：

- 账号、登录态、手机号验证码与资料字段。
- 配对关系、加入请求、关系类型切换和解绑流程数据。
- 每日记录、情绪标签、互动频率、图片分析结果、语音转写结果。
- 日报、周报、月报、趋势结果和关系智能结论。
- 关系任务、手动任务、反馈评分和执行状态。
- 时间轴归档、干预计划、叙事对齐和风险等级。
- 隐私审计、删除申请、保留治理状态。

存储方式如下：

- 结构化数据：开发环境可使用 SQLite，部署环境推荐 PostgreSQL。
- 临时与风控状态：推荐使用 Redis。
- 原始附件：写入上传目录，默认通过后端签名地址访问。

### 3.3 测试与回归数据

仓库当前主要保留可重复使用的测试样本和验证脚本，用于持续验证系统能力。

代表目录如下：

- `tmp/voice-regression/`：语音情绪和转写回归样本。
- `backend/.test-dbs/`：运行测试时生成的临时数据库目录。
- `output/`：真实环境冒烟、前端错误探针、手机端双账号、端口矩阵和轻压测脚本。

使用原则：

- 这类数据主要用于验证和复盘。
- 不建议混入正式业务统计。
- 一次性截图、日志和导出件建议在复核完成后清理，避免仓库持续膨胀。

### 3.4 交付与展示产物

仓库默认只保留可重复运行的验证脚本与少量回归样本。

这部分内容的定位是：

- 需要对外展示时，可临时重新生成截图、日志和校验结果。
- 这类内容不作为业务主数据源，不参与模型训练口径。
- 复核完成后，应优先保留脚本而不是长期堆积导出产物。

## 4. 数据边界

为避免误用，建议统一按以下口径理解和管理项目数据：

1. 演示样例仅用于演示、培训和页面验证，不映射真实用户身份。
2. 本地开发数据库仅用于联调和手工验证，不作为生产数据备份。
3. `tmp/` 下默认只建议保留可重复使用的回归样本，其余临时数据库和导出件应及时清理。
4. `output/` 中的脚本属于工程验证工具，运行生成的截图、日志和导出件不作为正式业务统计口径。
5. 未经过额外合规审查前，不应将仓库中的数据整体用于外部共享、商业转售或模型训练。

## 5. 安全与合规要求

当前实现已体现以下数据安全约束：

- 上传目录默认不通过 Nginx 直接公网暴露。
- 文件访问默认依赖签名接口，而非公共静态链接。
- 隐私中心支持查看状态、审计记录和删除申请。
- 生产环境会校验弱密钥、弱数据库密码和调试开关，避免带着不安全配置启动。

交付前建议执行一次数据清理：

1. 清理 `tmp/` 下不再需要的测试数据库、导出件和临时音频。
2. 检查本地重新生成的截图或日志中是否包含不适合外发的调试信息。
3. 确认演示邮箱、昵称、邀请码仍为示例值。
4. 不将 `.env`、生产密钥和真实用户上传文件直接打包提交。

## 6. 推荐对外表述

如需在申报书、交付说明或答辩材料中简要描述本项目数据资产，可使用以下表述：

> 本项目当前使用的数据资产主要包括三部分：一是前端内置的人工编排演示样例数据，用于产品演示和界面验证；二是开发、测试与回归阶段生成的本地数据库、语音样本和归档验证文件，用于工程验证；三是系统运行时产生的关系记录、报告结果、任务状态和隐私审计数据，按数据库与签名文件访问机制进行管理。当前仓库默认不包含面向外部公开发布的真实用户训练语料。
